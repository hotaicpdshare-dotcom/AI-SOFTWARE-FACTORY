# Data Directory

## raw

原始資料。保留來源與追溯性，不修改、不覆寫，也不預設提交 GitHub。若含公司敏感資料，應留在受控環境。

## sample

匿名化、虛構、小量或經核准的安全資料，可供 AI、開發與測試使用。可以提交，但必須確認不含秘密、個資或公司機密。

## processed

清洗與轉換後供產品使用的資料。必須由版本化且可重複執行的腳本重新產生，並保留輸入、輸出、版本與驗證狀態。

## Data Flow

```text
raw -> sample -> processed
```

不要直接在 raw 上清洗，也不要把未經確認的 processed 結果當成業務事實。
