# Acceptance Criteria

## Functional

- [ ] Core user flow works.
- [ ] Required functions match the confirmed scope.

## Data Accuracy

- [ ] KPI and calculation results trace to confirmed definitions.
- [ ] Missing, duplicate, invalid and partial data behavior is documented.

## Usability

- [ ] Primary users can complete the intended task clearly.
- [ ] Empty, loading and error states are understandable.

## Performance

- [ ] Expected data volume and response time are acceptable.
- [ ] Large or long-running work does not block the intended workflow.

## Security

- [ ] Access and permissions are enforced where required.
- [ ] No secrets, credentials or sensitive data are exposed or committed.

## Responsive Design

- [ ] Desktop behavior is acceptable.
- [ ] Mobile behavior is acceptable where applicable.

## Error Handling

- [ ] Invalid input has clear feedback.
- [ ] Failed operations do not silently lose or duplicate records.

## Deployment

- [ ] Deployment steps and environment requirements are documented.
- [ ] Rollback procedure is known.

## User Acceptance

- [ ] User review or UAT is complete.
- [ ] Known limitations are accepted or tracked for follow-up.
